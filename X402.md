# x402 en Prisma

## ¿Qué es x402?

**x402** es un protocolo de pago HTTP que extiende el estándar HTTP 402 "Payment Required". Permite cobrar por API usando cripto (USDC en este caso) sin pasar por pasarelas de pago tradicionales. El cliente firma una transferencia EIP-3009, el facilitador valida y liquida, y la API responde con el recurso solicitado.

## ¿Dónde se usa x402 en Prisma?

### Backend

| Archivo | Función |
|---------|---------|
| `src/x402/expressSetup.js` | Configura el middleware de pago: red (Fuji/C-Chain), monto USDC, facilitador PayAI, ruta protegida `GET /api/analyze/:address` |
| `src/createApp.js` | Aplica `createX402PaymentMiddleware()` cuando `X402_ENABLED=true`; expone headers `PAYMENT-RESPONSE`, `X-PAYMENT-RESPONSE` |
| `src/controllers/agent.controller.js` | El agente autónomo usa x402 para pagar la API cuando hace llamadas a `/api/analyze` |

### Frontend

| Archivo | Función |
|---------|---------|
| `ledgerlens-front/src/lib/api.ts` | Si hay `walletClient` conectado, usa `wrapFetchWithPayment` para firmar pago USDC cuando el backend responde 402; extrae `payment_tx_hash` del header `PAYMENT-RESPONSE` |
| `ledgerlens-front/src/features/analysis/hooks/useRunAnalysis.ts` | Con wallet conectada, cambia a Fuji/C-Chain (red de cobro) y pasa `walletClient` a `fetchAnalysis` |
| `ledgerlens-front/src/components/layout/DashboardLayout.tsx` | Muestra badge "x402 · Pago USDC" y enlace al hash cuando `payment_tx_hash` existe; mensaje informativo sobre x402 en EmptyState |

## Flujo de pago

1. **Sin x402** (`X402_ENABLED=false`): El análisis es gratuito. Cualquiera puede llamar a `/api/analyze/:address` sin wallet.

2. **Con x402** (`X402_ENABLED=true`):
   - El backend protege `GET /api/analyze/:address` con el middleware x402.
   - Primera llamada sin pago → el servidor responde **402 Payment Required** con los requisitos (monto USDC, red, payTo).
   - El frontend, si tiene `walletClient` (wallet conectada), usa `wrapFetchWithPayment` para:
     - Leer el 402
     - Crear el payload de pago (EIP-3009) con el signer
     - Firmar y enviar los headers `PAYMENT-SIGNATURE` / `X-PAYMENT`
     - Reintentar la petición; el facilitador valida, liquida USDC y responde 200 con el análisis
   - El header `PAYMENT-RESPONSE` trae el hash de la tx de pago → se guarda en `payment_tx_hash` y se muestra en la UI.

## Variables de entorno

| Variable | Descripción |
|----------|-------------|
| `X402_ENABLED` | `true` para activar cobro por API |
| `X402_PAY_TO` | Dirección que recibe el USDC (obligatoria si x402 está activo) |
| `X402_AMOUNT_USDC` | Monto en unidades base (6 decimales); ej. `10000` = 0.01 USDC |
| `X402_NETWORK` | `fuji` o `mainnet` / `avalanche` (eip155:43113 o eip155:43114) |
| `X402_FACILITATOR_URL` | URL del facilitador (por defecto PayAI) |
| `VITE_X402_NETWORK` | En el frontend: red a la que cambiar la wallet para pagar (`fuji` o `mainnet`) |

## Resumen visual

```
Usuario sin wallet + X402_ENABLED=true
  → GET /api/analyze → 402
  → Error: "Conecta tu wallet en la red configurada"

Usuario con wallet conectada (Fuji/C-Chain) + X402_ENABLED=true
  → GET /api/analyze → 402
  → wrapFetchWithPayment: firma USDC → reintenta
  → Facilitador valida → 200 + PAYMENT-RESPONSE (tx hash)
  → Dashboard muestra badge "x402 · Pago USDC"
```
